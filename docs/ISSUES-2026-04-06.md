## BLOCKER (5건)

### B-1. 자기참조 순환 의존성

- **파일:** `package.json:88`
- **현상:** `dependencies`에 `"tenetx": "^5.0.1"` 포함
- **영향:** npm 레지스트리 첫 publish 시 404 (자기 자신을 찾으려 함). 설치 시 패키지 이중 설치.
- **수정:** `dependencies`에서 `"tenetx": "^5.0.1"` 삭제
- **상태:** [x]

### B-2. postinstall fixOwnership 경로 오류

- **파일:** `scripts/postinstall.js:756`
- **현상:** `fixOwnership(join(HOME, '.claude'), join(HOME, '.compound'))` — 실제 데이터는 `~/.tenetx/`
- **영향:** `sudo npm i -g tenetx` 시 `~/.tenetx/`가 root 소유로 남아 EACCES 에러
- **수정:** `'.compound'` → `'.tenetx'`
- **상태:** [x]

### B-3. HOOK_REGISTRY 모듈 레벨 파싱 crash

- **파일:** `scripts/postinstall.js:354`
- **현상:** `const HOOK_REGISTRY = JSON.parse(readFileSync(...))` — `main()` 밖 top-level, try/catch 없음
- **영향:** `hooks/hook-registry.json` 없으면 전체 postinstall crash → `npm install` 실패
- **수정:** try/catch 감싸기, fallback `[]`
- **상태:** [x]

### B-4. runLifecycleCheck() SessionStart hook timeout 초과

- **파일:** `src/hooks/session-recovery.ts:383`
- **현상:** `runLifecycleCheck(sessionId)` 동기 실행. 내부에서 솔루션 파일당 `execFileSync('grep')` 호출 (현재 37개). stdin 대기 2s + lifecycle 3-14s → hook timeout 3s 초과
- **영향:** SessionStart hook 매일 1회 timeout → recovery 메시지 전부 드롭, compound 미실행
- **수정:** detached background spawn으로 분리
- **상태:** [x]

### B-5. context-guard Stop 이벤트에서 잘못된 hookEventName

- **파일:** `src/hooks/context-guard.ts:92`
- **현상:** `approveWithContext(..., 'UserPromptSubmit')` — Stop 이벤트 핸들러에서 UserPromptSubmit 이벤트 이름 사용
- **영향:** Claude Code가 이벤트 이름 불일치로 `additionalContext` 무시 → context limit 경고 메시지가 Claude에게 전달되지 않음
- **수정:** `approveWithWarning()` 사용 (Stop 이벤트에서 지원되는 방식)
- **상태:** [x]

## WARNING — Windows 비호환 (3건)

- **파일:** `src/core/spawn.ts:~43`, `src/hooks/session-recovery.ts:357`
- **현상:** `new URL(import.meta.url).pathname` → Windows에서 `/C:/Users/...` 형식 반환 (잘못된 경로)
- **영향:** Windows에서 auto-compound-runner 경로 해석 실패 → compound 미실행
- **수정:** `fileURLToPath(import.meta.url)` 사용

- **현상:** `execFileSync('sleep', ['3'])` — Windows에 sleep 바이너리 없음
- **수정:** `Atomics.wait(new Int32Array(new SharedArrayBuffer(4)), 0, 0, 3000)`
- **상태:** [x]

### W-W3. build 스크립트 POSIX 전용

- **파일:** `package.json` build script
- **현상:** `rm -rf dist && ... && chmod +x ...` — Windows 미지원
