### 시각화 라이브러리 (Intiviz)

- 스쿼드별로 ECharts/Chart.js 기반 구현이 제각각이라 스타일 일관성과 확장성이 떨어지고, 커스텀 차트 요구가 생길 때마다 화면 단위로 다시 구현해야 하는 비용이 컸습니다. 이를 줄이기 위해 3~4개 제품에서 공통 사용하는 D3 + TypeScript 기반의 framework-agnostic 시각화 라이브러리를 설계·구현했습니다.
- React/Vue 공용으로 사용할 수 있는 Headless 구조와 Atom/Molecule 2-Layer API를 설계해 고수준/저수준 커스터마이징을 모두 지원했고, 16개 draw primitive와 4개 chart component를 기반으로 Bar/Line/Area/Donut/Radar 등 8종 차트와 Composite Chart까지 확장했습니다.
- 옵션 증가로 생긴 Props Explosion은 도메인별 config 구조와 TypeScript utility type으로 정리해, 새로운 시각화 요구도 공통 primitive와 config 조합으로 수용할 수 있는 기반으로 바꿨습니다.

### Module Federation 기반 마이크로 프론트엔드

- Vue 3 중심 구조로 인한 채용·유지보수 부담이 커졌지만, 전체를 한 번에 React로 옮기기엔 비용과 위험이 큰 상황이었습니다. iframe도 보안 제약으로 어려워, Vite + Federation 기반으로 Vue 3 host에서 BI 기능을 React remote app 2개로 분리하는 구조를 설계했습니다.
- 리모트가 Provider 트리를 자체 보유하도록 구성해 host/remote 경계에서 런타임과 상태를 격리했고, 그 결과 점진적 React 전환과 독립 배포가 가능한 경로를 만들었습니다.
- remoteEntry 캐싱으로 구버전 리모트가 로드되는 문제는 no-cache 배포와 CloudFront invalidation을 CI/CD에 연결해 해결했고, 전면 재작성 없이 React 전환을 시작할 수 있는 현실적인 구조를 증명했습니다.

### 전환 실행 / 개발 생산성

- React 전환 방향을 제안하는 데서 끝내지 않고, 페이지 단위 공존 전략과 React 버전 공통 모듈, 전환 가이드를 정리해 팀이 실제로 옮겨갈 수 있는 기준을 만들었습니다.
- AI 도구와 워크플로우를 실무에서 검증해 공유했고, 사내 지식 하네스와 E2E 자동 생성 도구를 만들어 좋은 방법이 개인 노하우로 끝나지 않고 팀 자산으로 축적되도록 했습니다.
